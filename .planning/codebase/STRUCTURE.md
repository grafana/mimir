## Directory Purposes

**`cmd/`:**
- Purpose: `main` packages, one per binary. Thin wrappers that parse flags and hand off to `pkg/`.
- Contains: `cmd/<binary>/main.go`, `Dockerfile`, occasionally `help*.txt.tmpl` or config-descriptor outputs.
- Key files: `cmd/mimir/main.go` (config load + start), `cmd/mimir/config-descriptor.json` (flag metadata generated by `make doc`), `cmd/mimirtool/main.go`.

**`pkg/mimir/`:**
- Purpose: The composition root. Instantiates every service, wires the dskit module dependency graph.
- Key files: `pkg/mimir/mimir.go` (`Config`, `Mimir`, `New`), `pkg/mimir/modules.go` (all `init<Module>` methods + `setupModuleManager`), `pkg/mimir/runtime_config.go` (hot-reload of overrides), `pkg/mimir/sanity_check.go` (startup validation).

**`pkg/<service>/` (one per service):**
- Purpose: One Go package per Mimir service. The package name usually matches the directory name.
- Service packages: `distributor`, `ingester`, `querier`, `frontend`, `scheduler`, `storegateway`, `compactor`, `ruler`, `alertmanager`, `blockbuilder`, `usagetracker`.
- Each typically contains:
  - `<service>.go` — the top-level struct and `New<Service>` constructor.
  - `<service>_ring.go` — ring config and token generation.
  - `<service>_http.go` — HTTP handlers for admin/status endpoints.
  - `errors.go` — error types and gRPC status translation.
  - `<service>.proto` / `<service>.pb.go` — gRPC API (distributor, ingester, ruler, scheduler, alertmanager).
  - `*_test.go` — co-located unit tests (Go convention).

**`pkg/storage/`:**
- Purpose: Storage abstractions shared by ingester, querier, store-gateway, compactor, ruler, alertmanager.
- Subdirs:
  - `pkg/storage/bucket/` — generic object-store client with backends `s3/`, `gcs/`, `azure/`, `swift/`, `filesystem/`. Wraps `thanos-io/objstore`.
  - `pkg/storage/tsdb/` — Mimir-specific TSDB glue: `block/` (block meta/marker/fetcher), `bucketindex/` (tenant index), `indexcache/`, `bucketcache/`, `config.go`.
  - `pkg/storage/ingest/` — Kafka producer/consumer for ingest-storage mode. See `DESIGN.md`, `INTERNALS.md` inside.
  - `pkg/storage/chunk/`, `pkg/storage/series/`, `pkg/storage/lazyquery/`, `pkg/storage/sharding/` — query-side helpers.

**`pkg/util/`:**
- Purpose: Cross-cutting utilities. Anything shared across services.
- Subdirs:
  - `pkg/util/validation/` — `Limits`, `Overrides`, blocked/limited queries, separate metrics.
  - `pkg/util/validation/exporter/` — overrides-exporter service implementation.
  - `pkg/util/globalerror/` — stable user-facing error IDs.
  - `pkg/util/log/` — package `util_log`, go-kit logger setup.
  - `pkg/util/spanlogger/` — tracing-integrated logger.
  - `pkg/util/pool/`, `pkg/util/requestbuffers.go` — buffer pooling.
  - `pkg/util/limiter/`, `pkg/util/reactivelimiter/` — rate/inflight limiters.
  - `pkg/util/activitytracker/`, `pkg/util/shutdownmarker/` — operational helpers.
  - `pkg/util/test/`, `pkg/util/testkafka/` — test fixtures.

**`pkg/mimirpb/`:**
- Purpose: Protobuf types used across services (`WriteRequest`, `TimeSeries`, `Labels`, `Sample`, `MetricMetadata`).
- Key files: `pkg/mimirpb/mimir.proto`, `mimir.pb.go` (generated), `timeseries.go` (helpers), `compat.go` (conversion to/from Prometheus types), `custom_pools.go` (pool helpers), `validate.go`.

**`pkg/api/`:**
- Purpose: Shared HTTP router. Each service's routes get registered here via `API.Register<Service>`.
- Key files: `pkg/api/api.go` (`API`, `New`, `Register*`), `pkg/api/tenant.go` (tenant-validation middleware), `pkg/api/handlers.go`, `pkg/api/protobuf_codec.go`.

**`integration/`:**
- Purpose: End-to-end tests. Run Mimir in Docker containers and exercise real HTTP/gRPC paths.
- Key files: `integration/e2emimir/` (test helpers: `client.go`, `services.go`, `storage.go`), numerous `*_test.go` files per feature. Tests are gated by build tags (see `integration/README.md`).

**`operations/mimir/`:**
- Purpose: Jsonnet library to render Kubernetes manifests for production deployments. Published to `grafana/mimir/operations/mimir`.
- Key files: `mimir.libsonnet` (entry point), per-service `*.libsonnet` (`distributor.libsonnet`, `ingester.libsonnet`, ...), `config.libsonnet`, `jsonnetfile.json`.

**`operations/mimir-mixin/`:**
- Purpose: Grafana monitoring mixin — dashboards, recording rules, alerts.
- Key files: `mixin.libsonnet`, `dashboards/`, `alerts/`, `recording_rules.libsonnet`, `config.libsonnet`.
- Rendered output committed to `operations/mimir-mixin-compiled/`, `mimir-mixin-compiled-baremetal/`, `mimir-mixin-compiled-gem/`.

**`operations/helm/`:**
- Purpose: Helm chart for Kubernetes deployment.
- Key files: `operations/helm/charts/mimir-distributed/` (the chart), `operations/helm/tests/`, `operations/helm/scripts/`, `operations/helm/CLAUDE.md` (chart-specific conventions).

**`development/`:**
- Purpose: Docker-compose stacks for running Mimir locally in each deployment mode.
- Key files: `development/mimir-microservices-mode/compose-up.sh`, `development/mimir-monolithic-mode/`, `development/mimir-ingest-storage/` (Kafka stack), `development/common/`.

**`docs/`:**
- Purpose: Documentation source. Published via Grafana's Hugo-based toolkit.
- Key files: `docs/sources/mimir/` (main docs tree), `docs/sources/helm-charts/` (chart docs), `docs/proposals/` (design proposals). Architecture references live in `docs/sources/mimir/references/architecture/`. Runbooks in `docs/sources/mimir/manage/mimir-runbooks/_index.md`.

**`tools/`:**
- Purpose: One-off Go tools and shell scripts for build, CI, bucket ops, debugging.
- Examples: `tools/doc-generator/` (renders config reference), `tools/config-inspector/`, `tools/copyblocks/`, `tools/split-file-verify/`, `tools/github-next-pr-number.sh`, `tools/benchmark-query-engine/`.

**`vendor/`:**
- Purpose: Vendored Go dependencies. The repo uses `go mod vendor` and commits the `vendor/` tree.
- Generated: Yes (`go mod tidy && go mod vendor`).
- Committed: Yes — changes must be committed alongside `go.mod` and `go.sum` per `CONTRIBUTING.md`.

## Key File Locations

**Entry Points:**
- `cmd/mimir/main.go`: The mimir server binary.
- `cmd/mimirtool/main.go`: The `mimirtool` CLI.
- `cmd/query-tee/main.go`: The query-tee differential-testing binary.
- `cmd/metaconvert/main.go`: Bucket metadata converter.

**Configuration:**
- `pkg/mimir/mimir.go`: root `Config` struct (`Config.RegisterFlags`).
- `pkg/util/validation/limits.go`: per-tenant `Limits` (overridable at runtime).
- `pkg/mimir/runtime_config.go`: runtime overrides loader.
- `docs/configurations/`: sample config files.
- `cmd/mimir/config-descriptor.json`: structured description of every flag (generated).

**Core Logic:**
- `pkg/mimir/modules.go`: service wiring and dependency graph.
- `pkg/distributor/distributor.go`: write path.
- `pkg/ingester/ingester.go`, `pkg/ingester/ingester_push.go`, `pkg/ingester/ingester_query.go`: ingest and query on ingester.
- `pkg/querier/querier.go`, `pkg/querier/blocks_store_queryable.go`, `pkg/querier/distributor_queryable.go`: read path.
- `pkg/frontend/v2/frontend.go`, `pkg/frontend/querymiddleware/`: query-frontend.
- `pkg/scheduler/scheduler.go`, `pkg/scheduler/queue/queue.go`: query scheduling.
- `pkg/storegateway/gateway.go`, `pkg/storegateway/bucket.go`, `pkg/storegateway/bucket_stores.go`: historical block serving.
- `pkg/compactor/compactor.go`, `pkg/compactor/split_merge_compactor.go`: compaction.
- `pkg/ruler/ruler.go`, `pkg/ruler/manager.go`: rule evaluation.
- `pkg/alertmanager/multitenant.go`: multi-tenant Alertmanager.
- `pkg/streamingpromql/engine.go`: Mimir Query Engine.

**Testing:**
- Unit tests: co-located `_test.go` next to source (e.g. `pkg/distributor/distributor_test.go`).
- Integration tests: `integration/*_test.go` with helpers in `integration/e2emimir/`.
- Test fixtures: `pkg/*/fixtures/`, `pkg/util/test/`, `pkg/util/testkafka/`.

## Naming Conventions

**Files:**
- `snake_case.go`. Pattern: `<component>_<concern>.go`. Examples: `ingester_push.go`, `bucket_compactor.go`, `blocks_store_queryable.go`.
- Test files: `<name>_test.go` co-located in the same package.
- Generated protobuf: `<name>.pb.go` (alongside the `.proto`). Some packages also have `.pb.go.expdiff` (expected-diff files for codegen review).
- HTML templates for admin pages: `<component>.gohtml` (e.g. `pkg/compactor/status.gohtml`, `pkg/storegateway/blocks.gohtml`).

**Directories:**
- `pkg/<service>/` for services (one package per service).
- `pkg/<service>/client/` for the gRPC client library consumed by other services (e.g. `pkg/ingester/client/`).
- `pkg/<service>/<servicepb>/` for generated protobuf packages (e.g. `pkg/distributor/distributorpb/`, `pkg/scheduler/schedulerpb/`, `pkg/alertmanager/alertmanagerpb/`).
- `pkg/*/fixtures/` for test fixtures.

**Packages:**
- Package name matches directory name. Where the directory has hyphens or underscores, the package usually drops them (e.g. `storegateway` is dir name and package name).
- Common import aliases used project-wide (enforced by `make format`): `util_log "github.com/grafana/mimir/pkg/util/log"`, `mimir_tsdb "github.com/grafana/mimir/pkg/storage/tsdb"`, `ingester_client "github.com/grafana/mimir/pkg/ingester/client"`, `prom_storage "github.com/prometheus/prometheus/storage"`.

## Where to Add New Code

**New service module:**
- Primary code: `pkg/<service>/` — create `<service>.go` with `Config`, struct, `New<Service>` constructor, and `starting/running/stopping` lifecycle.
- Wire into process: add constant + `initFoo` method in `pkg/mimir/modules.go`, register in `setupModuleManager`, add to `deps` map.
- HTTP/gRPC routes: register in `pkg/api/api.go` via a new `RegisterFoo` method.
- Tests: `pkg/<service>/<service>_test.go` co-located.

**New write path (new protocol for ingestion):**
- Primary handler: `pkg/distributor/<protocol>.go` + `<protocol>_test.go`. Follow the pattern of `pkg/distributor/otel.go` or `pkg/distributor/influx.go`.
- Route registration: `pkg/api/api.go:RegisterDistributor`.
- Feed into existing `Distributor.PushWithMiddlewares`.

**New query middleware:**
- `pkg/frontend/querymiddleware/<name>.go` + test. Implement `Middleware` interface. Wire into the pipeline in `pkg/frontend/querymiddleware/roundtrip.go`.

**New storage backend:**
- `pkg/storage/bucket/<backend>/` — implement `thanos-io/objstore.Bucket`. Register in `pkg/storage/bucket/client.go`.

**New per-tenant limit:**
- Add field to `pkg/util/validation/limits.go`. Register flag in `RegisterFlags`. Add accessor method on `Overrides`. Consumers call `overrides.<Limit>(tenantID)`.
- Consider runtime override (YAML) vs. startup flag. Document in `docs/sources/mimir/configure/about-versioning.md` if stability-tier relevant.

**New error:**
- Per `CLAUDE.md`: add constant to `pkg/util/globalerror/user.go`. Create a helper in the same package to build the message. Add runbook entry under `docs/sources/mimir/manage/mimir-runbooks/_index.md`.

**New CLI flag / config field:**
- Add to the relevant `Config` struct, register in the package's `RegisterFlags`. Then run `make reference-help doc` and commit the regenerated config docs.

**New jsonnet deployment config:**
- `operations/mimir/<name>.libsonnet`. Run `make format-jsonnet-manifests`.

**New dashboard/alert:**
- `operations/mimir-mixin/dashboards/<name>.libsonnet` or `operations/mimir-mixin/alerts/<name>.libsonnet`. Run `make format-mixin` then `make build-mixin` to update compiled outputs.

**New helm template:**
- `operations/helm/charts/mimir-distributed/templates/`. See `operations/helm/CLAUDE.md`.

## Special Directories

**`vendor/`:**
- Purpose: Vendored copies of every direct and transitive Go dependency.
- Generated: Yes (`go mod vendor`).
- Committed: Yes. Must be updated together with `go.mod`/`go.sum`.

**`operations/mimir-mixin-compiled*/`:**
- Purpose: Rendered YAML output of the mixin jsonnet (for users who don't run jsonnet themselves).
- Generated: Yes (`make build-mixin`).
- Committed: Yes.

**`cmd/mimir/help.txt.tmpl`, `cmd/mimir/help-all.txt.tmpl`, `cmd/mimir/config-descriptor.json`:**
- Purpose: CLI help text + machine-readable flag metadata, driving docs and the config-inspector tool.
- Generated: Yes (`make doc` / `make reference-help`).
- Committed: Yes.

**`pkg/*/*.pb.go`, `pkg/*/*.pb.go.expdiff`:**
- Purpose: protobuf-generated Go. `.expdiff` captures expected drift from raw codegen for human-reviewed patches.
- Generated: Yes (`make protos`).
- Committed: Yes.

**`mimir-build-image/`:**
- Purpose: Dockerfile + helper scripts for the containerized build environment used by `make` targets.
- Contains: Go toolchain pin, goimports, protoc, jsonnet, shellcheck, etc.

**`.claude/skills/`:**
- Purpose: Project-local Claude skills. Currently contains `split-file` (procedure for splitting a large Go file while preserving git-follow history).
