# Phase 1 Plan 02: WellFormedVerifier Summary

**Concrete BlockVerifier wrapping pkg/storage/tsdb/block.VerifyBlock with Deep (checkChunks=true) and Medium (checkChunks=false) modes, proven against real GenerateBlockFromSpec blocks corrupted four ways.**

## Accomplishments

- `WellFormedVerifier` struct with `NewWellFormedVerifier(logger log.Logger, mode Mode) *WellFormedVerifier`. `Name()` returns the stable string `"well-formed"`.
- `Verify(ctx, blockDir, meta)` does an early `ctx.Err()` check (cheap between-block cancellation) then delegates to `block.VerifyBlock(ctx, v.logger, blockDir, meta.MinTime, meta.MaxTime, v.checkChunks)`, wrapping any error with `"well-formed check failed: %w"` so failures are self-identifying in the Report.
- Four test cases lock the intended mode classification against real TSDB blocks generated by `block.GenerateBlockFromSpec` in `t.TempDir()`:
  - `TestWellFormedVerifier_ValidBlock` — Deep and Medium both pass.
  - `TestWellFormedVerifier_TruncatedChunk` — truncating chunks/000001 to 10 bytes: Deep fails, error message contains "well-formed check failed".
  - `TestWellFormedVerifier_MangledIndex` — zeroing the first 100 bytes of `index`: Deep and Medium both fail.
  - `TestWellFormedVerifier_ChecksumMismatch_DeepFailsMediumPasses` — flipping one bit at offset 10 (first chunk's data region): Deep fails, Medium passes.
- Shared test helpers (`sampleAt`, `generateValidBlock`, `corruptChunkSegment`, `mangleIndex`, `flipChunkByte`) live in `testhelper_test.go` and are reusable by Plan 01-03's `singleutcday_test.go` in the same package.
- `sampleAt` returns `testutil.Sample{TS: ts, Val: v}` using the canonical `github.com/grafana/mimir/pkg/util/test` Sample type — no `prom_testutil` (red herring), no hand-rolled stub struct.
- All 10 tests in the package pass on first run: `go test ./pkg/mimirtool/backfill/verify/... -count=1` → ok (0.224s). The six Wave 1 framework tests are unchanged.

## Decisions Made

- **Stable check name.** `Name()` returns `"well-formed"` as a constant string literal. This is the string log-line consumers (Plan 01-05 CLI) and Report-reading code (Plan 01-04) will grep for, so it's locked here.
- **Error wrapping string.** Chose `"well-formed check failed: %w"` (lowercase, idiomatic Go error-wrap style). `TestWellFormedVerifier_TruncatedChunk` asserts on the substring `"well-formed check failed"`, pinning the contract.
- **Flip offset = 10.** Deterministic offset inside the first chunk's data region: `SegmentHeaderSize=8` + uvarint length (1 byte for any chunk under 128 bytes) + encoding byte (1 byte) = 10. Any future change that moves Prometheus's chunk segment layout will break this one test — that's the intended tripwire, not an accidental coupling.
- **`testutil.Sample{TS, Val}` lock.** Explicitly called out in `sampleAt`'s doc comment and in this summary's frontmatter. Plan 01-04's integration tests in `pkg/mimirtool/client/` will re-declare the same helper inline (Go's `_test.go` helpers don't cross package boundaries) and must use the same `testutil.Sample{TS: …, Val: …}` shape to stay consistent.

## Deviations from Plan

### Code Deviations

**1. [Rule 1 — Bug] Checksum-mismatch flip offset: 256 → 10**

- **Found during:** Task 2 (first `go test` run).
- **Issue:** The plan's action text picks `flipChunkByte(t, dir, 256)` for `TestWellFormedVerifier_ChecksumMismatch_DeepFailsMediumPasses`. With 5 XOR-encoded float samples the resulting `chunks/000001` file is well under 256 bytes, so the helper's `ReadAt` returned `io.EOF` and the test failed during setup, not during Verify.
- **Fix:** Changed the offset to `10` — the first byte past the 8-byte segment header + 1-byte uvarint length + 1-byte encoding byte, so the flip lands on the first chunk's first data byte regardless of chunk size. Expanded the test comment to explain the layout arithmetic so future readers can see why offset 10 is safe.
- **Files modified:** `pkg/mimirtool/backfill/verify/wellformed_test.go`
- **Verification:** `go test ./pkg/mimirtool/backfill/verify/... -run TestWellFormedVerifier_ChecksumMismatch -count=1` exits 0; Deep fails with a CRC-related error, Medium returns nil (Medium doesn't open the chunks directory at all).
- **Committed in:** `eca683551d` (Task 2 commit — the fix is baked into the test as written).
