<objective>
Ship the first batch-level verifier: `DuplicateDayVerifier`. Implements the `BatchVerifier` seam that Phase 1 shipped but never exercised in production. The check buckets blocks by UTC-day number (key = `meta.MinTime / msPerDay`), collects any bucket with more than one block, and returns an error enumerating every colliding day with its ULIDs.

Purpose: Covers CHECK-04. Prevents users from uploading two blocks covering the same UTC day in a single invocation, which would create guaranteed compaction conflicts.

Output: One production file + one test file. Green under `go test ./pkg/mimirtool/backfill/verify/... -count=1`.

### Locked design (from planning context — DO NOT re-debate)

1. **Type:** `DuplicateDayVerifier struct { logger log.Logger }`
2. **Constructor:** `NewDuplicateDayVerifier(logger log.Logger) *DuplicateDayVerifier`
3. **`Name() string`** returns `"duplicate-day"`
4. **`Verify(ctx context.Context, blocks []BlockRef) error`:**
   - Build `dayToULIDs := map[int64][]string{}` where key = `blk.Meta.MinTime / msPerDay`
   - Iterate blocks, append each `blk.Meta.ULID.String()` to the bucket
   - Collect day keys where `len(ulids) > 1` into `collisions`
   - If `len(collisions) == 0`, return nil
   - Else sort collision day keys ascending; within each collision, sort ULIDs lexicographically; return a single formatted error
5. **Error format:** `"multiple blocks cover the same UTC day: day %d: [ULID1, ULID2]; day %d: [ULID3, ULID4, ULID5]"` — days in ascending order, ULIDs lexicographic within each day. Use `strings.Join` and `fmt.Fprintf(&buf, ...)` or `fmt.Errorf`.
6. **Half-open caveat:** Blocks that passed the per-block `SingleUTCDayVerifier` are already confined to one UTC day, so `meta.MinTime / msPerDay` is unambiguous. If a block slips that check (e.g. user registered `DuplicateDayVerifier` without `SingleUTCDayVerifier`), we key on `MinTime`'s day only — a block spanning two days will collide only on its `MinTime` day. Document this in the file comment and do NOT add a defensive re-check: splitting one bad block across two day buckets would mask genuine single-day collisions.
7. **yoloString safety:** ULIDs come from `meta.ULID.String()` which is a `ulid.ULID` (fixed-size 16-byte array) -> allocates a fresh string. Safe to retain across the function return. Document this reasoning in the file comment.
</objective>

   ```go
   func (v *DuplicateDayVerifier) Verify(_ context.Context, blocks []BlockRef) error {
       if len(blocks) < 2 {
           return nil
       }
       dayToULIDs := make(map[int64][]string, len(blocks))
       for _, blk := range blocks {
           day := blk.Meta.MinTime / msPerDay
           dayToULIDs[day] = append(dayToULIDs[day], blk.Meta.ULID.String())
       }
       // Collect colliding days (more than one block).
       var collidingDays []int64
       for day, ulids := range dayToULIDs {
           if len(ulids) > 1 {
               collidingDays = append(collidingDays, day)
           }
       }
       if len(collidingDays) == 0 {
           return nil
       }
       sort.Slice(collidingDays, func(i, j int) bool { return collidingDays[i] < collidingDays[j] })
       var parts []string
       for _, day := range collidingDays {
           ulids := dayToULIDs[day]
           sort.Strings(ulids)
           parts = append(parts, fmt.Sprintf("day %d: [%s]", day, strings.Join(ulids, ", ")))
       }
       return fmt.Errorf("multiple blocks cover the same UTC day: %s", strings.Join(parts, "; "))
   }
   ```
